# 🏋️ מעקב אימונים - מדריך עדכון גרסאות

## 📋 תהליך עדכון גרסה חדשה

### שלב 1️⃣: עדכון מספר הגרסה

ערוך את הקובץ `src/version.py`:

```python
__version__ = "1.1.0"  # שנה את מספר הגרסה
__release_date__ = "2025-10-25"  # עדכן תאריך

# הוסף לרשימת השינויים
CHANGELOG = {
    "1.1.0": [
        "תיקון באג X",
        "הוספת פיצ'ר Y",
        # ...
    ],
    "1.0.0": [
        # גרסה קודמת...
    ]
}
```

### שלב 2️⃣: עריכת הקוד

1. ערוך את הקבצים הדרושים ב-`src/`
2. בדוק שהכל עובד: `python src/app.py`
3. בדוק שאין שגיאות: `python -m py_compile src/app.py`

### שלב 3️⃣: בניית EXE חדש

הפעל את סקריפט הבנייה:

```bash
# בפעם הראשונה - ודא ש-PyInstaller מותקן
.venv\Scripts\pip install pyinstaller

# בנה את ה-EXE
.venv\Scripts\python build.py
```

או ידנית:

```bash
.venv\Scripts\pyinstaller --clean workout_tracker.spec
```

### שלב 4️⃣: בדיקת ה-EXE

1. הקובץ יהיה ב-`dist\מעקב_אימונים.exe`
2. הרץ אותו ובדוק שהכל עובד
3. בדוק שמספר הגרסה מופיע בכותרת החלון

### שלב 5️⃣: הפצה

1. העתק את הקובץ `מעקב_אימונים.exe` למקום הרצוי
2. שתף אותו עם משתמשים
3. הם יכולים להחליף את ה-EXE הישן בחדש
4. **הנתונים שלהם יישמרו!** (קבצי JSON נפרדים)

## 📊 מדיניות גרסאות (Semantic Versioning)

```
X.Y.Z
│ │ └─ PATCH: תיקוני באגים קטנים
│ └─── MINOR: פיצ'רים חדשים (תואם אחורה)
└───── MAJOR: שינויים משמעותיים (לא תואם אחורה)
```

### דוגמאות:

- `1.0.0` → `1.0.1` - תיקון באג
- `1.0.1` → `1.1.0` - הוספת תרגיל חדש
- `1.9.0` → `2.0.0` - שינוי מבנה קבצי נתונים
